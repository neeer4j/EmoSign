=====================================================
WHAT IS THIS APP?
=====================================================

A Python application that watches your hands through 
your webcam and translates sign language into text.

Works with:
- Live webcam feed
- Uploaded video files
- Supports American Sign Language (ASL)
- Now with voice output, themes, and lots more!


=====================================================
⚠️ BEFORE YOU START - INSTALL THESE FIRST! ⚠️
=====================================================

STEP 1: Make sure you have Python installed
-----------------------------------------
You need Python 3.8 or higher.
Check by typing in terminal: python --version

If not installed, download from: https://python.org


STEP 2: Install all required packages
------------------------------------
Open terminal/command prompt in the project folder and run:

    pip install -r requirements.txt

This will install:
- PySide6 (for the app windows and buttons)
- opencv-python (for camera and image processing)
- mediapipe (for hand tracking)
- scikit-learn (for machine learning)
- numpy (for math calculations)
- pyttsx3 (for voice/speech output)
- pyperclip (for copy/paste)
- reportlab (for PDF export)
- torch (PyTorch - for neural network classifiers)


STEP 3: Download MediaPipe models (if missing)
---------------------------------------------
The app needs these files in the "models/" folder:
- hand_landmarker.task
- face_landmarker.task

If missing, download from:
https://storage.googleapis.com/mediapipe-models/


STEP 4: Run the app!
-------------------
    python main.py

=====================================================
CORE FEATURES
=====================================================

hand tracking & recognition:
- tracks your hands in real-time (uses mediapipe)
- now supports TWO-HAND gestures (for signs that need both hands)
- tells you what letter you're signing (uses random forest ml model)
- DUAL NEURAL NETWORK system: static FFN for still gestures, LSTM for motion gestures
- auto-detects whether hand is moving or still and picks the right model
- reads your face too and tells you if you look happy, sad, angry etc
- hybrid mode: recognizes both letters AND dynamic gestures at the same time

video file support:
- supports mp4, avi, mov, mkv, webm
- tracks hands properly now (no jittery mess) because we used video mode
- lower thresholds for videos so it catches blurry hands better
- has fallback geometry detection if the ML model gets confused

translation engine:
- translates sentences: type out full sentences by signing letter by letter
- three modes: instant (each letter), word (builds words), sentence (builds sentences)
- auto-translate when you pause signing
- saves your history so you can see what you said later

CORE FOLDER (core/) - The Brain
-------------------------------
core/__init__.py
    Just marks this as a Python package.

core/interpretation_engine.py
    MAIN: The new orchestrator engine that coordinates sentence-level translation.
    - Handles gesture detection, temporal buffering, and sentence mapping
    - Supports multiple modes: LIVE_CAMERA, VIDEO_FILE, TEXT_TO_SIGN
    - Provides callbacks for UI integration and partial match hints

core/gesture_dictionary.py
    Central dictionary of all gestures and predefined sentence mappings.
    - 26 letters (A-Z) + 10 numbers (0-9) for fingerspelling
    - 30+ word-level gestures (HELLO, GOODBYE, YES, NO, THANK_YOU, etc.)
    - 50+ predefined sentence mappings (greeting, response, question, etc.)
    - Maps gesture sequences to English sentences

core/sequence_buffer.py
    Temporal gesture accumulation with noise filtering and deduplication.
    - 5-frame stability threshold for gesture confirmation
    - Automatically detects gesture boundaries
    - 2.5s inactivity timeout for automatic translation
    - Prevents duplicate gestures from being added consecutively

core/sentence_mapper.py
    Maps accumulated gesture sequences to English sentences.
    - Matches sequences to predefined sentence patterns
    - Falls back to fingerspelling when no pattern matches
    - Bidirectional: Also includes TextToSignMapper for reverse translation

core/sign_visualizer.py
    Generates text-to-sign visualizations with step-by-step instructions.
    - Creates SignVisualization objects with gesture steps
    - Returns gesture names and descriptions for each step

core/pipeline.py
    Legacy pipeline controller (maintained for compatibility).
    - Connects camera → detection → translation

core/gesture_sequence.py
    Stores information about each gesture (what sign, how confident, when).

core/voice_output.py
    Makes the computer speak the translations out loud.

core/export_manager.py
    Saves translations to files (PDF, text, subtitles, etc.).

core/analytics.py
    Tracks how much you use the app and gives achievement badges.

ML FOLDER (ml/) - The Learning
------------------------------
ml/__init__.py
    Just marks this as a Python package.

ml/classifier.py
    Legacy brain - Random Forest model that recognizes signs.

ml/static_classifier.py
    Static FFN (Feedforward Neural Network) classifier.
    Uses a deep residual NN (512→512→256→128→64) on full 87-feature set
    (63 normalised coords + z-depth + thumb distances + curl angles + orientation).
    Handles still/stationary hand gestures (A-Z letters).
    Trained with PyTorch. Best accuracy: 100%.

ml/dynamic_classifier.py
    Dynamic LSTM classifier for motion-based gestures.
    Uses a 2-layer LSTM (hidden=256) on sequences of 30 frames (30×87 features).
    Each frame uses the full FeatureExtractor output for maximum accuracy.
    Handles gestures that involve movement (J, Z, WAVE, etc.).
    Trained with PyTorch. Best accuracy: ~99%.

ml/dual_model_manager.py
    The smart router that picks which model to use each frame.
    Measures hand displacement between frames:
    - Low displacement → static FFN (single frame prediction)
    - High displacement for 5+ frames → buffers 30 frames → LSTM prediction
    Returns (label, confidence, model_used) for every frame.

ml/trainer.py
    Teaches the legacy Random Forest model using training data.

ml/heuristic_classifier.py
    Backup detector using simple rules (if finger up = this letter).

ml/data_collector.py
    Collects hand data for training new gestures.

ml/gesture_accumulator.py
    Combines multiple frames into one stable gesture.

ml/personalized_trainer.py
    Trains the model specifically for YOUR hand shapes.

=====================================================
TROUBLESHOOTING
=====================================================

"Camera not found"
→ Check if another app is using your camera
→ Try changing CAMERA_ID in config.py (0, 1, or 2)

"Module not found" error
→ Run: pip install -r requirements.txt

"Hand not detected"
→ Make sure your hand is well-lit
→ Keep your hand within camera view
→ Try adjusting confidence in Settings

"Voice not working"
→ Check if pyttsx3 is installed
→ Try: pip install pyttsx3

App crashes on start
→ Make sure all dependencies are installed
→ Check if models/ folder has the required files

=====================================================
DEV NOTES
=====================================================

architecture changes (v3.1):
- complete redesign of interpretation engine with modular components
- new core modules: gesture_dictionary, sequence_buffer, sentence_mapper, interpretation_engine, sign_visualizer
- uses constrained vocabulary approach (50+ predefined sentences + fingerspelling fallback)
- gesture sequences mapped to English sentences (e.g., HELLO + YOU + HOW → "Hello, how are you?")
- temporal aggregation with 5-frame stability and 2.5s inactivity timeout
- explicit scope acknowledgment: NOT full ASL grammar, uses English word order
- maintains backward compatibility with legacy pipeline

architecture changes (v3.2 - Dual Neural Network):
- added static FFN classifier (ml/static_classifier.py) - deep residual NN
  trained on full 87-feature set from FeatureExtractor (coords + derived features)
  includes z-depth diffs, thumb-finger distances, curl angles, orientation
- added dynamic LSTM classifier (ml/dynamic_classifier.py) - 2-layer LSTM (hidden=256)
  trained on 30-frame sequences with 87 features per frame
- added DualModelManager (ml/dual_model_manager.py) - movement-aware routing
  that auto-switches between static and dynamic models based on hand displacement
- movement detection threshold: 0.015 normalised, 5 consecutive frames to trigger LSTM
- both models saved as PyTorch .pth files in models/ directory
- integrated into CameraWidget and VideoPlayerWidget via nn_gesture_detected signal
- existing heuristic + rule-based systems remain fully operational alongside NNs
- no UI changes required - new signal can be consumed by pages as needed

architecture changes (v3.3 - Accuracy Overhaul):
- static FFN upgraded from 63 raw coords to full 87-feature FeatureExtractor output
- FFN architecture deepened: input→512→512→256→128→64 with residual skip connections,
  BatchNorm, and Dropout for better gradient flow and 26-class discrimination
- dynamic LSTM upgraded from 63 to 87 features per frame, hidden_size increased to 256
- training data generator improved with letter-specific adjustments to disambiguate
  similar hand shapes (U/V/H/R spread, R finger-crossing z-depth, G/Q rotation, etc.)
- AdamW optimizer + cosine annealing + label smoothing + gradient clipping
- static FFN: 100% test accuracy (26 classes), dynamic LSTM: ~99% test accuracy

interpreter design patterns:
- SignLanguageEngine: Main orchestrator with callback system
- TemporalSequenceBuffer: Handles gesture accumulation and deduplication
- SentenceMapper: Pattern matching with fallback to fingerspelling
- GestureDictionary: Centralized storage of gestures and sentence mappings
- SignVisualizer: Text-to-sign visualization engine

dual neural network design:
- StaticFFN: 3-layer feedforward NN (256→128→64→classes) with BatchNorm + Dropout
- DynamicLSTM: 2-layer LSTM (hidden=128) → FC (64→classes) with Dropout
- DualModelManager: Movement-aware router with frame buffering
  * computes mean normalised landmark displacement each frame
  * displacement < 0.015 → StaticFFN on single frame (~1ms inference)
  * displacement >= 0.015 for 5+ frames → buffer 30 frames → LSTM (~5ms inference)
  * when motion stops mid-buffer, tries to classify partial sequence
  * returns (label, confidence, model_used) for each frame
- Models are saved separately: static_ffn_model.pth, dynamic_lstm_model.pth
- Both loaded at startup via DualModelManager.load()
